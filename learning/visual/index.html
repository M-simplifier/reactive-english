Assurance-driven development

ADD は「全部を証明する」ではなく、境界ごとに一番効く保証を選ぶ。

Type

TypeSystem

API payload、UI action、設定値など、配線ミスをコンパイル時に潰す。

例: schema bridge、App.UiAction、DatabaseBackend
EBT

Example-based tests

HTTP、SQLite、OAuth、DOM action のような境界を具体例で叩く。

例: dev login、401、vocabulary review、Google tokeninfo
PBT

Property-based tests

入力空間の広い純粋関数を、独立した参照モデルと比較する。

例: mastery、review spacing、shuffle permutation
Proof

Formal proof

小さく安定した意味論だけを Lean で証明する。

例: bounded mastery、monotone buckets、auth claim acceptance

Three technical lenses

この repo で特に学ぶ価値が高い3つの構造。

1. FRP は小さく使う

`Event` は発火、`Behavior` は現在値、`Network` は cleanup を持つ。 ここでは高階FRP研究をせず、SPA runtime の message flow に集中している。

DOM actionMsgreducerrender + command

2. スキーマ境界を先に固定する

`SchemaBridge/Spec.hs` から Haskell と PureScript の型を生成する。 これで frontend/backend の payload ずれが、推測ではなく機械的に見える。

Spec.hsGenerated.hsGenerated.purs

3. 本番公開は鍵を置かない

GitHub Actions は OIDC で GCP に短命認証する。DB URL は Secret Manager、 Terraform state は秘密扱い、Cloud Run は小さい上限で動かす。

GitHubOIDCWIFCloud Run

Authentication

Google認証とdev導線は、同じ session 形へ収束する。

実Googleログインは最小回数で確認し、複数ユーザー検証は dev login で高速化する。 重要なのは「外部 provider の曖昧なwire shape」をそのまま信頼しないことです。

  • `email_verified` の Bool/String を normalize して fail closed
  • ブラウザは HttpOnly session cookie を保持
  • study API は session なしなら 401

Vocabulary learning

語彙学習は、別プロダクト化せず lesson world に差し込む。

lexeme は5つの知識次元で進捗を持ち、recognition、meaning recall、 form recall、context、collocation を分けて扱う。

  • user + lexeme + dimension ごとに mastery を保存
  • review spacing と XP は純粋 kernel で検証
  • word mission は lesson attempt と独立した overlay

Infrastructure and CI/CD

公開フローは「手動deploy、短命認証、秘密はSecret Manager」が基本形。

01

Terraform apply

Artifact Registry、Cloud SQL、Secret Manager、IAM、WIF を作る。

02

GitHub Variables

project id や service account 名は repo variable。DB URL は置かない。

03

workflow_dispatch

通常 push では本番 deploy しない。Actions タブから手動実行。

04

OIDC auth

GitHub が WIF 経由で deployer service account を impersonate。

05

Docker image

PostgreSQL support 付きで build し、Artifact Registry へ push。

06

Cloud Run

Cloud SQL を attach し、DATABASE_URL を Secret Manager から注入。

月額放置の目安

$9-12

ほぼ Cloud SQL `db-f1-micro` + 10GB HDD。Cloud Run は `min-instances=0`。

攻撃時の上限ではない場所

request count、outbound transfer、logging は厳密な停止線ではない。

次の強化候補は Budget alert、Cloud Armor、ログ除外、自動停止。

停止手順

`terraform destroy` または GCP project 削除。Terraform state は秘密扱い。
